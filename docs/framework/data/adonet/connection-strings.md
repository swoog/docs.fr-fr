## <a name="connection-string-syntax"></a><span data-ttu-id="93cca-108">Syntaxe de chaîne de connexion</span><span class="sxs-lookup"><span data-stu-id="93cca-108">Connection string syntax</span></span>

<span data-ttu-id="93cca-109">Une chaîne de connexion est une liste délimitée par des points-virgules de paires clé/valeur de paramètre :</span><span class="sxs-lookup"><span data-stu-id="93cca-109">A connection string is a semicolon-delimited list of key/value parameter pairs:</span></span>
  
    keyword1=value; keyword2=value;
  
<span data-ttu-id="93cca-110">Mots clés ne respectent pas la casse.</span><span class="sxs-lookup"><span data-stu-id="93cca-110">Keywords are not case-sensitive.</span></span> <span data-ttu-id="93cca-111">Valeurs, toutefois, peuvent être la casse, en fonction de la source de données.</span><span class="sxs-lookup"><span data-stu-id="93cca-111">Values, however, may be case-sensitive, depending on the data source.</span></span> <span data-ttu-id="93cca-112">Peuvent contenir des mots clés et valeurs [blancs](https://en.wikipedia.org/wiki/Whitespace_character#Unicode).</span><span class="sxs-lookup"><span data-stu-id="93cca-112">Both keywords and values may contain [whitespace characters](https://en.wikipedia.org/wiki/Whitespace_character#Unicode).</span></span> <span data-ttu-id="93cca-113">Espaces de début et de fin est ignoré dans les mots clés et guillemets valeurs.</span><span class="sxs-lookup"><span data-stu-id="93cca-113">Leading and trailing white space is ignored in keywords and unquoted values.</span></span>

<span data-ttu-id="93cca-114">Si une valeur contient le point-virgule, [caractères de contrôle Unicode](https://en.wikipedia.org/wiki/Unicode_control_characters), ou à gauche ou un espace blanc, elle doit figurer entre guillemets simples ou doubles.</span><span class="sxs-lookup"><span data-stu-id="93cca-114">If a value contains the semicolon, [Unicode control characters](https://en.wikipedia.org/wiki/Unicode_control_characters), or leading or trailing white space, it must be enclosed in single or double quotation marks.</span></span> <span data-ttu-id="93cca-115">Exemple :</span><span class="sxs-lookup"><span data-stu-id="93cca-115">For example:</span></span>

    Keyword=" whitespace  ";
    Keyword='special;character';

<span data-ttu-id="93cca-116">Le caractère englobant n’a pas peut se produire au sein de la valeur qu’elle comprend.</span><span class="sxs-lookup"><span data-stu-id="93cca-116">The enclosing character may not occur within the value it encloses.</span></span> <span data-ttu-id="93cca-117">Par conséquent, une valeur contenant des guillemets simples peut être placées entre uniquement dans des guillemets doubles et vice versa :</span><span class="sxs-lookup"><span data-stu-id="93cca-117">Therefore, a value containing single quotation marks can be enclosed only in double quotation marks, and vice versa:</span></span>

    Keyword='double"quotation;mark';
    Keyword="single'quotation;mark";

<span data-ttu-id="93cca-118">Les guillemets eux-mêmes, ainsi que le signe égal, ne nécessite pas d’échappement, par conséquent, les chaînes de connexion suivantes sont valides :</span><span class="sxs-lookup"><span data-stu-id="93cca-118">The quotation marks themselves, as well as the equals sign, do not require escaping, so the following connection strings are valid:</span></span>

    Keyword=no "escaping" 'required';
    Keyword=a=b=c

<span data-ttu-id="93cca-119">Dans la mesure où chaque valeur est lue jusqu'à ce que le point-virgule suivant ou à la fin de chaîne, la valeur dans l’exemple de ce dernier est `a=b=c`, et le point-virgule final est facultatif.</span><span class="sxs-lookup"><span data-stu-id="93cca-119">Since each value is read till the next semicolon or the end of string, the value in the latter example is `a=b=c`, and the final semicolon is optional.</span></span>

<span data-ttu-id="93cca-120">Toutes les chaînes de connexion partagent la même syntaxe de base décrite ci-dessus.</span><span class="sxs-lookup"><span data-stu-id="93cca-120">All connection strings share the same basic syntax described above.</span></span> <span data-ttu-id="93cca-121">L’ensemble des mots clés reconnus varie selon le fournisseur, toutefois et a évolué au fil des années à partir d’API antérieures telles que *ODBC*.</span><span class="sxs-lookup"><span data-stu-id="93cca-121">The set of recognized keywords depends on the provider, however, and has evolved over the years from earlier APIs such as *ODBC*.</span></span> <span data-ttu-id="93cca-122">Le *.NET Framework* fournisseur de données pour *SQL Server* (`SqlClient`) prend en charge de nombreux mots clés à partir d’une API plus anciennes, mais se montre généralement plus souple et accepte des synonymes pour la plupart de la chaîne de connexion courantes mots clés.</span><span class="sxs-lookup"><span data-stu-id="93cca-122">The *.NET Framework* data provider for *SQL Server* (`SqlClient`) supports many keywords from older APIs, but is generally more flexible and accepts synonyms for many of the common connection string keywords.</span></span>

<span data-ttu-id="93cca-123">Fautes de frappe peut provoquer des erreurs.</span><span class="sxs-lookup"><span data-stu-id="93cca-123">Typing mistakes can cause errors.</span></span> <span data-ttu-id="93cca-124">Par exemple, `Integrated Security=true` est valide, mais `IntegratedSecurity=true` provoque une erreur.</span><span class="sxs-lookup"><span data-stu-id="93cca-124">For example, `Integrated Security=true` is valid, but `IntegratedSecurity=true` causes an error.</span></span>

<span data-ttu-id="93cca-125">Chaînes de connexion générées manuellement en cours d’exécution à partir de l’entrée utilisateur non validée sont vulnérables aux attaques par injection de chaîne et mettre en péril la sécurité au niveau de la source de données.</span><span class="sxs-lookup"><span data-stu-id="93cca-125">Connection strings constructed manually at run time from unvalidated user input are vulnerable to string-injection attacks and jeopardize security at the data source.</span></span> <span data-ttu-id="93cca-126">Pour résoudre ces problèmes, *ADO.NET* 2.0 a introduit [générateurs de chaînes de connexion](../../../../docs/framework/data/adonet/connection-string-builders.md) pour chaque *.NET Framework* fournisseur de données.</span><span class="sxs-lookup"><span data-stu-id="93cca-126">To address these problems, *ADO.NET* 2.0 introduced [connection string builders](../../../../docs/framework/data/adonet/connection-string-builders.md) for each *.NET Framework* data provider.</span></span> <span data-ttu-id="93cca-127">Ces générateurs de chaînes de connexion exposent des paramètres en tant que propriétés fortement typées et le rendent possible de valider la chaîne de connexion avant d’être envoyée à la source de données.</span><span class="sxs-lookup"><span data-stu-id="93cca-127">These connection string builders expose parameters as strongly-typed properties, and make it possible to validate the connection string before it's sent to the data source.</span></span>
