## <a name="Introducing"></a> <span data-ttu-id="62f41-107">Présentation de .NET Framework</span><span class="sxs-lookup"><span data-stu-id="62f41-107">What is the .NET Framework?</span></span>

- <span data-ttu-id="62f41-111">Gestion de la mémoire</span><span class="sxs-lookup"><span data-stu-id="62f41-111">Memory management.</span></span> <span data-ttu-id="62f41-112">Dans de nombreux langages de programmation, les programmeurs sont chargés d'allouer et de libérer la mémoire et de gérer les durées de vie des objets.</span><span class="sxs-lookup"><span data-stu-id="62f41-112">In many programming languages, programmers are responsible for allocating and releasing memory and for handling object lifetimes.</span></span> <span data-ttu-id="62f41-113">Dans les applications .NET Framework, le CLR fournit ces services de la part de l’application.</span><span class="sxs-lookup"><span data-stu-id="62f41-113">In .NET Framework apps, the CLR provides these services on behalf of the app.</span></span>

- <span data-ttu-id="62f41-114">Système de type commun.</span><span class="sxs-lookup"><span data-stu-id="62f41-114">A common type system.</span></span> <span data-ttu-id="62f41-115">Dans les langages de programmation traditionnels, les types de base sont définis par le compilateur, ce qui complique l'interopérabilité interlangage.</span><span class="sxs-lookup"><span data-stu-id="62f41-115">In traditional programming languages, basic types are defined by the compiler, which complicates cross-language interoperability.</span></span> <span data-ttu-id="62f41-116">Dans le .NET Framework, les types de base sont définis par le système de type .NET Framework et sont communs à tous les langages qui ciblent le .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="62f41-116">In the .NET Framework, basic types are defined by the .NET Framework type system and are common to all languages that target the .NET Framework.</span></span>

- <span data-ttu-id="62f41-119">Frameworks et technologies de développement.</span><span class="sxs-lookup"><span data-stu-id="62f41-119">Development frameworks and technologies.</span></span> <span data-ttu-id="62f41-120">Le .NET Framework inclut des bibliothèques pour des domaines spécifiques de développement d’applications, tels qu’ASP.NET pour les applications web, ADO.NET pour l’accès aux données, Windows Communication Foundation pour les applications orientées service et Windows Presentation Foundation pour les applications bureau Windows.</span><span class="sxs-lookup"><span data-stu-id="62f41-120">The .NET Framework includes libraries for specific areas of app development, such as ASP.NET for web apps, ADO.NET for data access, Windows Communication Foundation for service-oriented apps, and Windows Presentation Foundation for Windows desktop apps.</span></span>

- <span data-ttu-id="62f41-121">Interopérabilité des langages.</span><span class="sxs-lookup"><span data-stu-id="62f41-121">Language interoperability.</span></span> <span data-ttu-id="62f41-122">Les compilateurs de langage qui ciblent le .NET Framework délivrent un code intermédiaire nommé Common Intermediate Language (CIL) qui est compilé à son tour au moment de l’exécution par le Common Language Runtime.</span><span class="sxs-lookup"><span data-stu-id="62f41-122">Language compilers that target the .NET Framework emit an intermediate code named Common Intermediate Language (CIL), which, in turn, is compiled at runtime by the common language runtime.</span></span> <span data-ttu-id="62f41-123">Avec cette fonctionnalité, les routines écrites dans un langage sont accessibles à d’autres langages pour que les programmeurs se concentrent sur la création d’applications dans leurs langages préférés.</span><span class="sxs-lookup"><span data-stu-id="62f41-123">With this feature, routines written in one language are accessible to other languages, and programmers focus on creating apps in their preferred languages.</span></span>

- <span data-ttu-id="62f41-124">Compatibilité des versions.</span><span class="sxs-lookup"><span data-stu-id="62f41-124">Version compatibility.</span></span> <span data-ttu-id="62f41-125">À de rares exceptions près, les applications qui sont développées avec une version particulière du .NET Framework s’exécutent sans modification sur une version ultérieure.</span><span class="sxs-lookup"><span data-stu-id="62f41-125">With rare exceptions, apps that are developed by using a particular version of the .NET Framework run without modification on a later version.</span></span>

- <span data-ttu-id="62f41-126">Exécution côte à côte.</span><span class="sxs-lookup"><span data-stu-id="62f41-126">Side-by-side execution.</span></span> <span data-ttu-id="62f41-127">Le .NET Framework aide à résoudre les conflits de versions en permettant à plusieurs versions du Common Langage Runtime de cohabiter sur le même ordinateur.</span><span class="sxs-lookup"><span data-stu-id="62f41-127">The .NET Framework helps resolve version conflicts by allowing multiple versions of the common language runtime to exist on the same computer.</span></span> <span data-ttu-id="62f41-128">Cela signifie que plusieurs versions d'applications peuvent coexister et qu’une application peut s’exécuter sur la version du .NET Framework avec laquelle elle a été générée.</span><span class="sxs-lookup"><span data-stu-id="62f41-128">This means that multiple versions of apps can coexist and that an app can run on the version of the .NET Framework with which it was built.</span></span> <span data-ttu-id="62f41-129">L’exécution côte-à-côte s’applique aux groupes de versions .NET Framework 1.0/1.1, 2.0/3.0/3.5 et 4/4.5.x/4.6.x/4.7.x/4.8.</span><span class="sxs-lookup"><span data-stu-id="62f41-129">Side-by-side execution applies to the .NET Framework version groups 1.0/1.1, 2.0/3.0/3.5, and 4/4.5.x/4.6.x/4.7.x/4.8.</span></span>

- <span data-ttu-id="62f41-130">Multiciblage.</span><span class="sxs-lookup"><span data-stu-id="62f41-130">Multitargeting.</span></span> <span data-ttu-id="62f41-131">En ciblant [.NET Standard](../../standard/net-standard.md), les développeurs créent des bibliothèques de classes qui fonctionnent sur plusieurs plateformes .NET Framework prises en charge par cette version de la norme.</span><span class="sxs-lookup"><span data-stu-id="62f41-131">By targeting [.NET Standard](../../standard/net-standard.md), developers create class libraries that work on multiple .NET Framework platforms supported by that version of the standard.</span></span> <span data-ttu-id="62f41-132">Par exemple, les bibliothèques qui ciblent .NET Standard 2.0 peuvent être utilisées par des applications qui ciblent .NET Framework 4.6.1, .NET Core 2.0 et UWP 10.0.16299.</span><span class="sxs-lookup"><span data-stu-id="62f41-132">For example, libraries that target the .NET Standard 2.0 can be used by apps that target the .NET Framework 4.6.1, .NET Core 2.0, and UWP 10.0.16299.</span></span>
