# <a name="how-to-map-hresults-and-exceptions"></a><span data-ttu-id="8aded-102">Comment : mapper des HRESULT et des exceptions</span><span class="sxs-lookup"><span data-stu-id="8aded-102">How to: Map HRESULTs and Exceptions</span></span>
<span data-ttu-id="8aded-103">Les méthodes COM signalent les erreurs en retournant des HRESULT ; les méthodes .NET les signalent en levant des exceptions.</span><span class="sxs-lookup"><span data-stu-id="8aded-103">COM methods report errors by returning HRESULTs; .NET methods report them by throwing exceptions.</span></span> <span data-ttu-id="8aded-104">Le runtime gère la transition entre les deux.</span><span class="sxs-lookup"><span data-stu-id="8aded-104">The runtime handles the transition between the two.</span></span> <span data-ttu-id="8aded-105">Chaque classe d’exception dans le .NET Framework est mappée à une valeur HRESULT.</span><span class="sxs-lookup"><span data-stu-id="8aded-105">Each exception class in the .NET Framework maps to an HRESULT.</span></span>  
  
 <span data-ttu-id="8aded-106">Les classes d’exceptions définies par l’utilisateur peuvent spécifier la valeur HRESULT appropriée.</span><span class="sxs-lookup"><span data-stu-id="8aded-106">User-defined exception classes can specify whatever HRESULT is appropriate.</span></span> <span data-ttu-id="8aded-107">Ces classes d’exceptions peuvent modifier dynamiquement la valeur HRESULT à retourner quand l’exception est générée par la définition du champ **HResult** sur l’objet exception.</span><span class="sxs-lookup"><span data-stu-id="8aded-107">These exception classes can dynamically change the HRESULT to be returned when the exception is generated by setting the **HResult** field on the exception object.</span></span> <span data-ttu-id="8aded-108">Des informations supplémentaires relatives à l’exception sont fournies au client par le biais de l’interface **IErrorInfo**, qui est implémentée sur l’objet .NET dans le processus non managé.</span><span class="sxs-lookup"><span data-stu-id="8aded-108">Additional information about the exception is provided to the client through the **IErrorInfo** interface, which is implemented on the .NET object in the unmanaged process.</span></span>  
  
 <span data-ttu-id="8aded-109">Si vous créez une classe qui étend **System.Exception**, vous devez définir le champ HRESULT pendant la construction.</span><span class="sxs-lookup"><span data-stu-id="8aded-109">If you create a class that extends **System.Exception**, you must set the HRESULT field during construction.</span></span> <span data-ttu-id="8aded-110">Sinon, la classe de base assigne la valeur HRESULT.</span><span class="sxs-lookup"><span data-stu-id="8aded-110">Otherwise, the base class assigns the HRESULT value.</span></span> <span data-ttu-id="8aded-111">Vous pouvez mapper de nouvelles classes d’exceptions à un HRESULT existant en fournissant la valeur dans le constructeur de l’exception.</span><span class="sxs-lookup"><span data-stu-id="8aded-111">You can map new exception classes to an existing HRESULT by supplying the value in the exception's constructor.</span></span>  
  
 <span data-ttu-id="8aded-112">Notez que le runtime ignore parfois un `HRESULT` si le thread comporte un `IErrorInfo`.</span><span class="sxs-lookup"><span data-stu-id="8aded-112">Note that the runtime will sometimes ignore an `HRESULT` in cases where there is an `IErrorInfo` present on the thread.</span></span>  <span data-ttu-id="8aded-113">Ce comportement peut se présenter si `HRESULT` et `IErrorInfo` ne représentent pas la même erreur.</span><span class="sxs-lookup"><span data-stu-id="8aded-113">This behavior can occur in cases where the `HRESULT` and the `IErrorInfo` do not represent the same error.</span></span>  
  
### <a name="to-create-a-new-exception-class-and-map-it-to-an-hresult"></a><span data-ttu-id="8aded-114">Pour créer une classe d’exception et la mapper à un HRESULT</span><span class="sxs-lookup"><span data-stu-id="8aded-114">To create a new exception class and map it to an HRESULT</span></span>  
  
1.  <span data-ttu-id="8aded-115">Utilisez le code suivant pour créer une classe d’exception appelée `NoAccessException` et mappez-la à la valeur HRESULT `E_ACCESSDENIED`.</span><span class="sxs-lookup"><span data-stu-id="8aded-115">Use the following code to create a new exception class called `NoAccessException` and map it to the HRESULT `E_ACCESSDENIED`.</span></span>  
  
    ```cpp  
    Class NoAccessException : public ApplicationException  
    {  
        NoAccessException () {  
        HResult = E_ACCESSDENIED;   
    }  
    }  
    CMyClass::MethodThatThrows  
    {  
    throw new NoAccessException();  
    }  
    ```
