 <span data-ttu-id="d2b27-104">L’héritage doit être linéaire, et non circulaire.</span><span class="sxs-lookup"><span data-stu-id="d2b27-104">Inheritance must be linear, not circular.</span></span> <span data-ttu-id="d2b27-105">Un type ne peut pas hériter d’un type qui hérite de celui-ci.</span><span class="sxs-lookup"><span data-stu-id="d2b27-105">A type cannot inherit from a type that inherits from it.</span></span>  
  
 <span data-ttu-id="d2b27-106">Un type ne peut pas non plus hériter d’un type qui n’est pas encore défini.</span><span class="sxs-lookup"><span data-stu-id="d2b27-106">A related restriction is that a type cannot inherit from a type that is not yet defined.</span></span> <span data-ttu-id="d2b27-107">L’héritage implique la possibilité de réutiliser des membres de la classe de base, qui exige à son tour que ces membres soient définis.</span><span class="sxs-lookup"><span data-stu-id="d2b27-107">Inheritance involves the ability to reuse members of the base class, which in turn requires that these members be defined.</span></span> <span data-ttu-id="d2b27-108">Par conséquent, Visual Basic ne peut pas compiler le code comme l’exemple suivant.</span><span class="sxs-lookup"><span data-stu-id="d2b27-108">Therefore, Visual Basic cannot compile code such as the following example.</span></span>

```  
    Inherits innerClass  
    Public Class innerClass  
        Public Sub doSomething()  
        End Sub  
    End Class  
End Class  
```
