# <a name="what-is-managed-code"></a><span data-ttu-id="56faf-103">Qu’est-ce que le code managé ?</span><span class="sxs-lookup"><span data-stu-id="56faf-103">What is "managed code"?</span></span>

<span data-ttu-id="56faf-104">Quand vous utilisez .NET Framework, vous rencontrez souvent le terme « code managé ».</span><span class="sxs-lookup"><span data-stu-id="56faf-104">When working with .NET Framework, you will often encounter the term "managed code".</span></span> <span data-ttu-id="56faf-105">Ce document explique ce que signifie ce terme et fournit des informations supplémentaires sur le sujet.</span><span class="sxs-lookup"><span data-stu-id="56faf-105">This document will explain what this term means and additional information around it.</span></span>

<span data-ttu-id="56faf-106">Pour faire simple, le code managé est du code dont l’exécution est gérée par un runtime.</span><span class="sxs-lookup"><span data-stu-id="56faf-106">To put it very simply, managed code is just that: code whose execution is managed by a runtime.</span></span> <span data-ttu-id="56faf-107">Dans ce cas, le runtime en question est appelé le **Common Language Runtime** ou CLR, indépendamment de l’implémentation ([Mono](https://www.mono-project.com/) ou .NET Framework, ou .NET Core).</span><span class="sxs-lookup"><span data-stu-id="56faf-107">In this case, the runtime in question is called the **Common Language Runtime** or CLR, regardless of the implementation ([Mono](https://www.mono-project.com/) or .NET Framework or .NET Core).</span></span> <span data-ttu-id="56faf-108">CLR est chargé de prendre le code managé, de le compiler en code machine, puis de l’exécuter.</span><span class="sxs-lookup"><span data-stu-id="56faf-108">CLR is in charge of taking the managed code, compiling it into machine code and then executing it.</span></span> <span data-ttu-id="56faf-109">Par ailleurs, le runtime fournit plusieurs services importants comme la gestion automatique de la mémoire, les limites de sécurité, la cohérence des types, etc.</span><span class="sxs-lookup"><span data-stu-id="56faf-109">On top of that, runtime provides several important services such as automatic memory management, security boundaries, type safety etc.</span></span>

<span data-ttu-id="56faf-110">Par opposition, le « code non managé » est la façon dont vous pouvez exécuter un programme C/C++.</span><span class="sxs-lookup"><span data-stu-id="56faf-110">Contrast this to the way you would run a C/C++ program, also called "unmanaged code".</span></span> <span data-ttu-id="56faf-111">Dans un environnement non managé, le programmeur est responsable de presque tout.</span><span class="sxs-lookup"><span data-stu-id="56faf-111">In the unmanaged world, the programmer is in charge of pretty much everything.</span></span> <span data-ttu-id="56faf-112">Le programme réel est, essentiellement, un fichier binaire que le système d’exploitation charge en mémoire et démarre.</span><span class="sxs-lookup"><span data-stu-id="56faf-112">The actual program is, essentially, a binary that the operating system (OS) loads into memory and starts.</span></span> <span data-ttu-id="56faf-113">Tout le reste, depuis la gestion de la mémoire aux considérations de sécurité, est à la charge du programmeur.</span><span class="sxs-lookup"><span data-stu-id="56faf-113">Everything else, from memory management to security considerations are a burden of the programmer.</span></span>

<span data-ttu-id="56faf-114">Le code managé est écrit dans un des langages de haut niveau qui peuvent être exécutés sur .NET, tels que C#, Visual Basic ou F#.</span><span class="sxs-lookup"><span data-stu-id="56faf-114">Managed code is written in one of the high-level languages that can be run on top of .NET, such as C#, Visual Basic, F# and others.</span></span> <span data-ttu-id="56faf-115">Quand vous compilez le code écrit dans ces langages avec leur compilateur respectif, vous n’obtenez pas de code machine.</span><span class="sxs-lookup"><span data-stu-id="56faf-115">When you compile code written in those languages with their respective compiler, you don’t get machine code.</span></span> <span data-ttu-id="56faf-116">Vous obtenez un code en **langage intermédiaire** que le runtime compile ensuite et exécute.</span><span class="sxs-lookup"><span data-stu-id="56faf-116">You get **Intermediate Language** code which the runtime then compiles and executes.</span></span> <span data-ttu-id="56faf-117">C++ est la seule exception à cette règle, car il peut également produire des fichiers binaires natifs, non managés qui s’exécutent sur Windows.</span><span class="sxs-lookup"><span data-stu-id="56faf-117">C++ is the one exception to this rule, as it can also produce native, unmanaged binaries that run on Windows.</span></span>

## <a name="intermediate-language--execution"></a><span data-ttu-id="56faf-118">Langage intermédiaire et exécution</span><span class="sxs-lookup"><span data-stu-id="56faf-118">Intermediate Language & execution</span></span>

<span data-ttu-id="56faf-119">Qu’est-ce que le « langage intermédiaire » (ou IL en abrégé) ?</span><span class="sxs-lookup"><span data-stu-id="56faf-119">What is "Intermediate Language" (or IL for short)?</span></span> <span data-ttu-id="56faf-120">Il s’agit d’un produit de compilation de code écrit dans des langages .NET de haut niveau.</span><span class="sxs-lookup"><span data-stu-id="56faf-120">It is a product of compilation of code written in high-level .NET languages.</span></span> <span data-ttu-id="56faf-121">Quand vous compilez votre code écrit dans un de ces langages, vous obtenez un fichier binaire constitué de langage intermédiaire.</span><span class="sxs-lookup"><span data-stu-id="56faf-121">Once you compile your code written in one of these languages, you will get a binary that is made out of IL.</span></span> <span data-ttu-id="56faf-122">Notez que le langage intermédiaire est indépendant de tout langage spécifique qui s’exécute sur le runtime. Il existe même une spécification distincte pour ce langage, que vous pouvez lire si vous le souhaitez.</span><span class="sxs-lookup"><span data-stu-id="56faf-122">It is important to note that the IL is independent from any specific language that runs on top of the runtime; there is even a separate specification for it that you can read if you’re so inclined.</span></span>

<span data-ttu-id="56faf-123">Une fois que vous avez produit du langage intermédiaire à partir de votre code de haut niveau, vous pouvez l’exécuter.</span><span class="sxs-lookup"><span data-stu-id="56faf-123">Once you produce IL from your high-level code, you will most likely want to run it.</span></span> <span data-ttu-id="56faf-124">C’est là que le CLR intervient et démarre le processus de compilation **juste-à-temps**, ou **JIT-ing** de votre code à partir du langage intermédiaire en code machine qui peut être exécuté sur un processeur.</span><span class="sxs-lookup"><span data-stu-id="56faf-124">This is where the CLR takes over and starts the process of **Just-In-Time** compiling, or **JIT-ing** your code from IL to machine code that can actually be run on a CPU.</span></span> <span data-ttu-id="56faf-125">De cette façon, le CLR sait exactement ce que votre code est en train de faire et peut donc le _gérer_ efficacement.</span><span class="sxs-lookup"><span data-stu-id="56faf-125">In this way, the CLR knows exactly what your code is doing and can effectively _manage_ it.</span></span>

<span data-ttu-id="56faf-126">Le langage intermédiaire est parfois appelé langage CIL (Common Intermediate Language) ou langage MSIL (Microsoft Intermediate Language).</span><span class="sxs-lookup"><span data-stu-id="56faf-126">Intermediate Language is sometimes also called Common Intermediate Language (CIL) or Microsoft Intermediate Language (MSIL).</span></span>

## <a name="unmanaged-code-interoperability"></a><span data-ttu-id="56faf-127">Interopérabilité du code non managé</span><span class="sxs-lookup"><span data-stu-id="56faf-127">Unmanaged code interoperability</span></span>

<span data-ttu-id="56faf-128">Bien entendu, le CLR permet de franchir les limites entre les environnements managé et non managé, et il y a beaucoup de code qui le fait, même dans les [bibliothèques de classes de base](framework-libraries.md).</span><span class="sxs-lookup"><span data-stu-id="56faf-128">Of course, the CLR allows passing the boundaries between managed and unmanaged world, and there is a lot of code that does that, even in the [Base Class Libraries](framework-libraries.md).</span></span> <span data-ttu-id="56faf-129">Cela s’appelle l’**interopérabilité** ou simplement **interop** en abrégé.</span><span class="sxs-lookup"><span data-stu-id="56faf-129">This is called **interoperability** or just **interop** for short.</span></span> <span data-ttu-id="56faf-130">Ces dispositions vous permettent, par exemple, d’encapsuler une bibliothèque non managée et d’y effectuer des appels.</span><span class="sxs-lookup"><span data-stu-id="56faf-130">These provisions would allow you to, for example, wrap up an unmanaged library and call into it.</span></span> <span data-ttu-id="56faf-131">Toutefois, notez qu’une fois que vous l’avez fait, quand le code franchit les limites du runtime, la gestion réelle de l’exécution est à nouveau entre les mains du code non managé et subit donc les mêmes restrictions.</span><span class="sxs-lookup"><span data-stu-id="56faf-131">However, it is important to note that once you do this, when the code passes the boundaries of the runtime, the actual management of the execution is again in the hand of unmanaged code, and thus falls under the same restrictions.</span></span>

<span data-ttu-id="56faf-132">De la même manière, C# est un langage qui vous permet d’utiliser des constructions non managées comme les pointeurs directement dans le code en utilisant ce que l’on appelle le **contexte unsafe**, qui désigne un élément de code dont l’exécution n’est pas gérée par le CLR.</span><span class="sxs-lookup"><span data-stu-id="56faf-132">Similar to this, C# is one language that allows you to use unmanaged constructs such as pointers directly in code by utilizing what is known as **unsafe context** which designates a piece of code for which the execution is not managed by the CLR.</span></span>
