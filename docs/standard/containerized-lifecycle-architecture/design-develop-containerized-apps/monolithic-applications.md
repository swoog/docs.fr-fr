## <a name="monolithic-application-deployed-as-a-container"></a><span data-ttu-id="f576a-145">Application monolithique déployée comme conteneur</span><span class="sxs-lookup"><span data-stu-id="f576a-145">Monolithic application deployed as a container</span></span>

<span data-ttu-id="f576a-146">Il existe des avantages à l’utilisation de conteneurs pour gérer les déploiements monolithiques.</span><span class="sxs-lookup"><span data-stu-id="f576a-146">There are benefits to using containers to manage monolithic deployments.</span></span> <span data-ttu-id="f576a-147">La mise à l’échelle des instances des conteneurs est beaucoup plus rapide et facile que le déploiement de machines virtuelles supplémentaires.</span><span class="sxs-lookup"><span data-stu-id="f576a-147">Scaling the instances of containers is far faster and easier than deploying additional VMs.</span></span>

<span data-ttu-id="f576a-148">Le déploiement de mises à jour comme images Docker est beaucoup plus rapide et efficace du point de vue du réseau.</span><span class="sxs-lookup"><span data-stu-id="f576a-148">Deploying updates as Docker images is far faster and network efficient.</span></span> <span data-ttu-id="f576a-149">Conteneurs docker démarrent généralement en secondes, ce qui accélère les déploiements.</span><span class="sxs-lookup"><span data-stu-id="f576a-149">Docker containers typically start in seconds, speeding rollouts.</span></span> <span data-ttu-id="f576a-150">Destruction d’un conteneur Docker est aussi simple que l’appel de la `docker stop` commande, normalement moins d’une seconde.</span><span class="sxs-lookup"><span data-stu-id="f576a-150">Tearing down a Docker container is as easy as invoking the `docker stop` command, typically completing in less than a second.</span></span>

<span data-ttu-id="f576a-151">Étant donné que les conteneurs sont immuables, par conception, vous devez jamais à vous soucier d’endommagement des machines virtuelles, car vous avez oublié par un script de mise à jour pour prendre en compte une configuration spécifique ou d’un fichier restant sur le disque.</span><span class="sxs-lookup"><span data-stu-id="f576a-151">Because containers are inherently immutable, by design, you never need to worry about corrupted VMs because an update script forgot to account for some specific configuration or file left on disk.</span></span>
