## <a name="a-simple-analogy"></a><span data-ttu-id="cb88b-141">Une simple analogie</span><span class="sxs-lookup"><span data-stu-id="cb88b-141">A simple analogy</span></span>

<span data-ttu-id="cb88b-142">Peut-être qu’une analogie simple peut aider à bien comprendre le concept de base de Docker.</span><span class="sxs-lookup"><span data-stu-id="cb88b-142">Perhaps a simple analogy can help getting the grasp of the core concept of Docker.</span></span>

<span data-ttu-id="cb88b-143">Retournons dans les années 1950 un instant.</span><span class="sxs-lookup"><span data-stu-id="cb88b-143">Let's go back in time to the 1950s for a moment.</span></span> <span data-ttu-id="cb88b-144">Ne portaient sur aucun processeur de word, et les photocopieurs ont été utilisés partout (Eh bien, type de).</span><span class="sxs-lookup"><span data-stu-id="cb88b-144">There were no word processors, and the photocopiers were used everywhere (well, kind of).</span></span>

<span data-ttu-id="cb88b-145">Imaginez que vous devez émettre rapidement des lots de lettres à envoyer aux clients, avec du papier et des enveloppes, et les livrer physiquement à l’adresse de chaque client (l’e-mail n’existait pas à l’époque).</span><span class="sxs-lookup"><span data-stu-id="cb88b-145">Imagine you're responsible for quickly issuing batches of letters as required, to mail them to customers, using real paper and envelopes, to be delivered physically to each customer's address (there was no email back then).</span></span>

<span data-ttu-id="cb88b-146">À un moment donné, vous réalisez que les lettres sont simplement une composition d’un grand ensemble de paragraphes, qui sont organisés en fonction des besoins, selon l’objectif de la lettre, donc vous concevez un système permettant d’émettre des lettres rapidement, afin d’obtenir une augmentation importante.</span><span class="sxs-lookup"><span data-stu-id="cb88b-146">At some point, you realize the letters are just a composition of a large set of paragraphs, which are picked and arranged as needed, according to the purpose of the letter, so you devise a system to issue letters quickly, expecting to get a hefty raise.</span></span>

<span data-ttu-id="cb88b-147">Le système est simple :</span><span class="sxs-lookup"><span data-stu-id="cb88b-147">The system is simple:</span></span>

1. <span data-ttu-id="cb88b-148">Vous commencez avec un jeu de feuilles transparentes contenant chacune un paragraphe.</span><span class="sxs-lookup"><span data-stu-id="cb88b-148">You begin with a deck of transparent sheets containing one paragraph each.</span></span>

2. <span data-ttu-id="cb88b-149">Pour émettre un ensemble de lettres, vous choisissez les feuilles avec les paragraphes dont vous avez besoin, vous les empilez et les alignez afin de pouvoir les lire correctement.</span><span class="sxs-lookup"><span data-stu-id="cb88b-149">To issue a set of letters, you pick the sheets with the paragraphs you need, then you stack and align them so they look and read fine.</span></span>

3. <span data-ttu-id="cb88b-150">Enfin, vous placez l’ensemble dans la photocopieuse, puis appuyez sur démarrer pour produire autant de lettres que nécessaire.</span><span class="sxs-lookup"><span data-stu-id="cb88b-150">Finally, you place the set in the photocopier and press start to produce as many letters as required.</span></span>

<span data-ttu-id="cb88b-151">C’est cela l’idée fondamentale de Docker (en version simplifiée).</span><span class="sxs-lookup"><span data-stu-id="cb88b-151">So, simplifying, that's the core idea of Docker.</span></span>

<span data-ttu-id="cb88b-152">Dans Docker, chaque couche est un jeu de résultats des modifications qui se produisent dans le système de fichiers après l’exécution d’une commande, par exemple, l’installation d’un programme.</span><span class="sxs-lookup"><span data-stu-id="cb88b-152">In Docker, each layer is the resulting set of changes that happen to the filesystem after executing a command, such as, installing a program.</span></span>

<span data-ttu-id="cb88b-153">Par conséquent, lorsque vous « regardez » le système de fichiers une fois que la couche a été copiée, vous voyez tous les fichiers, y compris la couche lorsque le programme a été installé.</span><span class="sxs-lookup"><span data-stu-id="cb88b-153">So, when you "look" at the filesystem after the layer has been copied, you see all the files, included the layer when the program was installed.</span></span>

<span data-ttu-id="cb88b-154">Vous pouvez considérer une image en tant que disque dur en lecture seule auxiliaire prêt à être installé dans un « ordinateur » où le système d’exploitation est déjà installé.</span><span class="sxs-lookup"><span data-stu-id="cb88b-154">You can think of an image as an auxiliary read-only hard disk ready to be installed in a "computer" where the operating system is already installed.</span></span>

<span data-ttu-id="cb88b-155">De même, vous pouvez considérer un conteneur comme « l’ordinateur » avec le disque dur de l’image installé.</span><span class="sxs-lookup"><span data-stu-id="cb88b-155">Similarly, you can think of a container as the "computer" with the image hard disk installed.</span></span> <span data-ttu-id="cb88b-156">Le conteneur, tout comme un ordinateur, peut être activé ou désactivé.</span><span class="sxs-lookup"><span data-stu-id="cb88b-156">The container, just like a computer, can be powered on or off.</span></span>
