<span data-ttu-id="7b407-118">Les [conteneurs Windows](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview) fonctionnent avec deux types de runtime :</span><span class="sxs-lookup"><span data-stu-id="7b407-118">To run [Windows Containers](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview), there are two types of runtimes:</span></span>

<span data-ttu-id="7b407-123">Les images de ces deux types de conteneurs sont créées et fonctionnent de la même façon.</span><span class="sxs-lookup"><span data-stu-id="7b407-123">The images for these containers are created the same way and function the same.</span></span> <span data-ttu-id="7b407-124">La différence réside dans la façon dont le conteneur est créé à partir de l’image, l’exécution d’un conteneur Hyper-V nécessite un paramètre supplémentaire.</span><span class="sxs-lookup"><span data-stu-id="7b407-124">The difference is in how the container is created from the image running a Hyper-V Container requires an extra parameter.</span></span> <span data-ttu-id="7b407-125">Pour plus d’informations, consultez [Conteneurs Hyper-V](https://docs.microsoft.com/virtualization/windowscontainers/manage-containers/hyperv-container).</span><span class="sxs-lookup"><span data-stu-id="7b407-125">For details, see [Hyper-V Containers](https://docs.microsoft.com/virtualization/windowscontainers/manage-containers/hyperv-container).</span></span>

<span data-ttu-id="7b407-139">Du fait que les conteneurs nécessitent beaucoup moins de ressources (par exemple, ils n’ont pas besoin d’un système d’exploitation complet), leur démarrage est rapide et leur déploiement est simple.</span><span class="sxs-lookup"><span data-stu-id="7b407-139">Because containers require far fewer resources (for example, they don't need a full OS), they're easy to deploy and they start fast.</span></span> <span data-ttu-id="7b407-140">Cela vous permet d’avoir une densité plus élevée, ce qui vous permet d’exécuter plus de services sur la même unité matérielle, limitant les coûts.</span><span class="sxs-lookup"><span data-stu-id="7b407-140">This allows you to have higher density, meaning that it allows you to run more services on the same hardware unit, thereby reducing costs.</span></span>

<span data-ttu-id="7b407-144">Une image conteneur est un moyen d’empaqueter une application ou un service et de les déployer ensuite d’une façon fiable et reproductible.</span><span class="sxs-lookup"><span data-stu-id="7b407-144">A container image is a way to package an app or service and deploy it in a reliable and reproducible way.</span></span> <span data-ttu-id="7b407-145">Plus qu’une simple technologie, Docker peut également être vu comme une philosophie et un processus.</span><span class="sxs-lookup"><span data-stu-id="7b407-145">You could say that Docker isn't only a technology but also a philosophy and a process.</span></span>

<span data-ttu-id="7b407-146">Les développeurs qui utilisent Docker ne disent pas : « Cela fonctionne sur ma machine, alors pourquoi pas en production ? »</span><span class="sxs-lookup"><span data-stu-id="7b407-146">When using Docker, you won't hear developers say, "It works on my machine, why not in production?"</span></span> <span data-ttu-id="7b407-147">Ils peuvent simplement dire « Cela s’exécute sur Docker », car l’application Docker empaquetée peut être exécutée sur n’importe quel environnement Docker pris en charge et elle s’exécute comme prévu sur toutes les cibles de déploiement (par exemple développement, assurance qualité, intermédiaire et production).</span><span class="sxs-lookup"><span data-stu-id="7b407-147">They can simply say, "It runs on Docker", because the packaged Docker application can be executed on any supported Docker environment, and it runs the way it was intended to on all deployment targets (such as Dev, QA, staging, and production).</span></span>

## <a name="a-simple-analogy"></a><span data-ttu-id="7b407-148">Une simple analogie</span><span class="sxs-lookup"><span data-stu-id="7b407-148">A simple analogy</span></span>

<span data-ttu-id="7b407-149">Peut-être qu’une analogie simple peut aider à bien comprendre le concept de base de Docker.</span><span class="sxs-lookup"><span data-stu-id="7b407-149">Perhaps a simple analogy can help getting the grasp of the core concept of Docker.</span></span>

<span data-ttu-id="7b407-150">Retournons dans les années 1950 un instant.</span><span class="sxs-lookup"><span data-stu-id="7b407-150">Let's go back in time to the 1950s for a moment.</span></span> <span data-ttu-id="7b407-151">Le traitement de texte n’existait pas et les photocopieurs étaient utilisés partout (ou presque).</span><span class="sxs-lookup"><span data-stu-id="7b407-151">There were no word processors, and the photocopiers were used everywhere (kind of).</span></span>

<span data-ttu-id="7b407-152">Imaginez que vous devez émettre rapidement des lots de lettres à envoyer aux clients, avec du papier et des enveloppes, et les livrer physiquement à l’adresse de chaque client (l’e-mail n’existait pas à l’époque).</span><span class="sxs-lookup"><span data-stu-id="7b407-152">Imagine you're responsible for quickly issuing batches of letters as required, to mail them to customers, using real paper and envelopes, to be delivered physically to each customer's address (there was no email back then).</span></span>

<span data-ttu-id="7b407-153">À un moment donné, vous réalisez que les lettres sont simplement une composition d’un grand ensemble de paragraphes, qui sont organisés en fonction des besoins, selon l’objectif de la lettre, donc vous concevez un système permettant d’émettre des lettres rapidement, afin d’obtenir une augmentation importante.</span><span class="sxs-lookup"><span data-stu-id="7b407-153">At some point, you realize the letters are just a composition of a large set of paragraphs, which are picked and arranged as needed, according to the purpose of the letter, so you devise a system to issue letters quickly, expecting to get a hefty raise.</span></span>

<span data-ttu-id="7b407-154">Le système est simple :</span><span class="sxs-lookup"><span data-stu-id="7b407-154">The system is simple:</span></span>

1. <span data-ttu-id="7b407-155">Vous commencez avec un jeu de feuilles transparentes contenant chacune un paragraphe.</span><span class="sxs-lookup"><span data-stu-id="7b407-155">You begin with a deck of transparent sheets containing one paragraph each.</span></span>

2. <span data-ttu-id="7b407-156">Pour émettre un ensemble de lettres, vous choisissez les feuilles avec les paragraphes dont vous avez besoin, vous les empilez et les alignez afin de pouvoir les lire correctement.</span><span class="sxs-lookup"><span data-stu-id="7b407-156">To issue a set of letters, you pick the sheets with the paragraphs you need, then you stack and align them so they look and read fine.</span></span>

3. <span data-ttu-id="7b407-157">Enfin, vous placez l’ensemble dans la photocopieuse, puis appuyez sur démarrer pour produire autant de lettres que nécessaire.</span><span class="sxs-lookup"><span data-stu-id="7b407-157">Finally, you place the set in the photocopier and press start to produce as many letters as required.</span></span>

<span data-ttu-id="7b407-158">C’est cela l’idée fondamentale de Docker (en version simplifiée).</span><span class="sxs-lookup"><span data-stu-id="7b407-158">So, simplifying, that's the core idea of Docker.</span></span>

<span data-ttu-id="7b407-159">Dans Docker, chaque couche est un jeu de résultats des modifications qui se produisent dans le système de fichiers après l’exécution d’une commande, par exemple, l’installation d’un programme.</span><span class="sxs-lookup"><span data-stu-id="7b407-159">In Docker, each layer is the resulting set of changes that happen to the filesystem after executing a command, such as, installing a program.</span></span>

<span data-ttu-id="7b407-160">Par conséquent, lorsque vous « regardez » le système de fichiers une fois que la couche a été copiée, vous voyez tous les fichiers, y compris la couche lorsque le programme a été installé.</span><span class="sxs-lookup"><span data-stu-id="7b407-160">So, when you "look" at the filesystem after the layer has been copied, you see all the files, included the layer when the program was installed.</span></span>

<span data-ttu-id="7b407-161">Vous pouvez considérer une image en tant que disque dur en lecture seule auxiliaire prêt à être installé dans un « ordinateur » où le système d’exploitation est déjà installé.</span><span class="sxs-lookup"><span data-stu-id="7b407-161">You can think of an image as an auxiliary read-only hard disk ready to be installed in a "computer" where the operating system is already installed.</span></span>

<span data-ttu-id="7b407-162">De même, vous pouvez considérer un conteneur comme « l’ordinateur » avec le disque dur de l’image installé.</span><span class="sxs-lookup"><span data-stu-id="7b407-162">Similarly, you can think of a container as the "computer" with the image hard disk installed.</span></span> <span data-ttu-id="7b407-163">Le conteneur, tout comme un ordinateur, peut être activé ou désactivé.</span><span class="sxs-lookup"><span data-stu-id="7b407-163">The container, just like a computer, can be powered on or off.</span></span>
