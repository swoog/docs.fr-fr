# <a name="docker-terminology"></a><span data-ttu-id="8d945-103">Terminologie Docker</span><span class="sxs-lookup"><span data-stu-id="8d945-103">Docker terminology</span></span>

<span data-ttu-id="8d945-104">Cette section liste les termes et les définitions que vous devez connaître avant d’explorer Docker de manière plus approfondie.</span><span class="sxs-lookup"><span data-stu-id="8d945-104">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="8d945-105">Pour obtenir d’autres définitions, consultez le [glossaire](https://docs.docker.com/glossary/) complet fourni par Docker.</span><span class="sxs-lookup"><span data-stu-id="8d945-105">For further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker .</span></span>

<span data-ttu-id="8d945-106">**Image conteneur (container image)** : package de toutes les dépendances et informations nécessaires pour créer un conteneur.</span><span class="sxs-lookup"><span data-stu-id="8d945-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="8d945-107">Une image inclut toutes les dépendances (notamment les frameworks) ainsi que la configuration de déploiement et d’exécution à utiliser par le runtime du conteneur.</span><span class="sxs-lookup"><span data-stu-id="8d945-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="8d945-108">En règle générale, une image est dérivée de plusieurs images de base qui sont empilées en couches pour former le système de fichiers du conteneur.</span><span class="sxs-lookup"><span data-stu-id="8d945-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="8d945-109">Une image est immuable une fois qu’elle a été créée.</span><span class="sxs-lookup"><span data-stu-id="8d945-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="8d945-110">**Conteneur (container)** : instance d’une image Docker.</span><span class="sxs-lookup"><span data-stu-id="8d945-110">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="8d945-111">Un conteneur représente l’exécution d’une application, d’un processus ou d’un service.</span><span class="sxs-lookup"><span data-stu-id="8d945-111">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="8d945-112">Il renferme une image Docker, un environnement d’exécution et un ensemble standard d’instructions.</span><span class="sxs-lookup"><span data-stu-id="8d945-112">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="8d945-113">Pour mettre un service à l’échelle, vous créez plusieurs instances d’un conteneur à partir de la même image.</span><span class="sxs-lookup"><span data-stu-id="8d945-113">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="8d945-114">Cela peut également être fait par un traitement par lots, qui passe des paramètres différents à chaque instance.</span><span class="sxs-lookup"><span data-stu-id="8d945-114">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="8d945-115">**Balise (tag)** : marque ou étiquette que vous pouvez appliquer aux images pour identifier les différentes images ou versions de l’image initiale (selon le numéro de version de l’environnement cible).</span><span class="sxs-lookup"><span data-stu-id="8d945-115">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="8d945-116">**Dockerfile** : fichier texte contenant des instructions pour la création d’une image Docker.</span><span class="sxs-lookup"><span data-stu-id="8d945-116">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="8d945-117">**Création (build)** : action de créer une image conteneur sur la base des informations et du contexte fournis par le fichier Dockerfile associé, plus des fichiers supplémentaires dans le dossier où l’image est créée.</span><span class="sxs-lookup"><span data-stu-id="8d945-117">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="8d945-118">Vous pouvez créer des images à l’aide de la commande docker build de Docker.</span><span class="sxs-lookup"><span data-stu-id="8d945-118">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="8d945-119">**Dépôt (repository)** : collection d’images Docker associées, identifiées par une balise qui indique la version de chaque image.</span><span class="sxs-lookup"><span data-stu-id="8d945-119">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="8d945-120">Certains dépôts contiennent plusieurs variantes d’une image spécifique, par exemple une image contenant des SDK (plus lourde), une image contenant uniquement des runtimes (plus légère), etc. Ces variantes peuvent être identifiées par des balises.</span><span class="sxs-lookup"><span data-stu-id="8d945-120">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="8d945-121">Un dépôt peut contenir des variantes de plateforme, comme une image Linux et une image Windows.</span><span class="sxs-lookup"><span data-stu-id="8d945-121">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="8d945-122">**Registre (registry)** : service qui fournit l’accès aux dépôts.</span><span class="sxs-lookup"><span data-stu-id="8d945-122">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="8d945-123">Le registre par défaut utilisé pour la plupart des images publiques est [Docker Hub](https://hub.docker.com/) (propriété de l’organisation Docker).</span><span class="sxs-lookup"><span data-stu-id="8d945-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="8d945-124">Un registre contient généralement des dépôts de plusieurs équipes.</span><span class="sxs-lookup"><span data-stu-id="8d945-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="8d945-125">Les entreprises utilisent souvent des registres privés pour stocker et gérer les images qu’elles ont créées.</span><span class="sxs-lookup"><span data-stu-id="8d945-125">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="8d945-126">Azure Container Registry est un autre exemple de registre.</span><span class="sxs-lookup"><span data-stu-id="8d945-126">Azure Container Registry is another example.</span></span>

<span data-ttu-id="8d945-127">**Docker Hub** : registre public dans lequel vous pouvez charger et manipuler des images.</span><span class="sxs-lookup"><span data-stu-id="8d945-127">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="8d945-128">Docker Hub fournit un hébergement d’images Docker, des registres publics ou privés, des déclencheurs de build et des webhooks, et l’intégration avec GitHub et Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="8d945-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="8d945-129">**Azure Container Registry** : ressource publique permettant d’utiliser des images Docker et leurs composants dans Azure.</span><span class="sxs-lookup"><span data-stu-id="8d945-129">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="8d945-130">Vous disposez ainsi d’un registre au plus près de vos déploiements dans Azure, qui vous permet de contrôler l’accès avec vos groupes et autorisations Azure Active Directory existants.</span><span class="sxs-lookup"><span data-stu-id="8d945-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="8d945-131">**Docker Trusted Registry (DTR)** : service de registre Docker (fourni par Docker) qui peut être installé en local pour résider au sein même du centre de données et du réseau de l’organisation.</span><span class="sxs-lookup"><span data-stu-id="8d945-131">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="8d945-132">Il est pratique pour les images privées devant être gérées en interne dans l’entreprise.</span><span class="sxs-lookup"><span data-stu-id="8d945-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="8d945-133">Docker Trusted Registry est intégré au produit Docker Datacenter.</span><span class="sxs-lookup"><span data-stu-id="8d945-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="8d945-134">Pour plus d’informations, consultez [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="8d945-134">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="8d945-135">**Docker Community Edition (CE)** : outils de développement pour Windows et macOS permettant de créer, d’exécuter et de tester des conteneurs localement.</span><span class="sxs-lookup"><span data-stu-id="8d945-135">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="8d945-136">Docker CE pour Windows fournit des environnements de développement pour les conteneurs Linux et Windows.</span><span class="sxs-lookup"><span data-stu-id="8d945-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="8d945-137">L’hôte Linux Docker sur Windows est basé sur une machine virtuelle [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx).</span><span class="sxs-lookup"><span data-stu-id="8d945-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="8d945-138">L’hôte pour les conteneurs Windows est directement basé sur Windows.</span><span class="sxs-lookup"><span data-stu-id="8d945-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="8d945-139">Docker CE pour Mac est basé sur le framework Hypervisor d’Apple et [l’hyperviseur xhyve](https://github.com/mist64/xhyve), qui fournit une machine virtuelle hôte Linux Docker sur Mac OS X. Docker CE pour Windows et pour Mac remplace Docker Toolbox, qui était basé sur Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="8d945-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="8d945-140">**Docker Enterprise Edition (EE)** : version pour l’entreprise qui fournit des outils Docker pour le développement Linux et Windows.</span><span class="sxs-lookup"><span data-stu-id="8d945-140">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="8d945-141">**Compose** : outil en ligne de commande et format de fichier YAML fournissant des métadonnées pour la définition et l’exécution d’applications multiconteneurs.</span><span class="sxs-lookup"><span data-stu-id="8d945-141">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="8d945-142">Vous définissez une application basée sur plusieurs images avec un ou plusieurs fichiers .yml qui peuvent remplacer les valeurs en fonction de l’environnement.</span><span class="sxs-lookup"><span data-stu-id="8d945-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="8d945-143">Après avoir créé les définitions, vous déployez l’application multiconteneurs entière à l’aide d’une seule commande (docker-compose up) qui crée un conteneur par image sur l’hôte Docker.</span><span class="sxs-lookup"><span data-stu-id="8d945-143">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="8d945-144">**Cluster** : collection d’hôtes Docker exposés en tant qu’hôte Docker virtuel unique, ce qui permet la mise à l’échelle de l’application en fonction du nombre d’instances des services répartis entre les différents hôtes au sein du cluster.</span><span class="sxs-lookup"><span data-stu-id="8d945-144">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="8d945-145">Vous pouvez créer des clusters Docker avec Docker Swarm, Mesosphere DC/OS, Kubernetes et Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="8d945-145">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="8d945-146">(Si vous utilisez Docker Swarm pour gérer un cluster, le cluster est généralement désigné sous le nom de *swarm*.)</span><span class="sxs-lookup"><span data-stu-id="8d945-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

<span data-ttu-id="8d945-147">**Orchestrator** : outil qui simplifie la gestion des clusters et des hôtes Docker.</span><span class="sxs-lookup"><span data-stu-id="8d945-147">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="8d945-148">Les orchestrateurs vous permettent de gérer les images, les conteneurs et les hôtes à l’aide d’une interface de ligne de commande (CLI) ou d’une interface graphique utilisateur.</span><span class="sxs-lookup"><span data-stu-id="8d945-148">Orchestrators enable you to manage their images, containers, and hosts through a command line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="8d945-149">Vous pouvez gérer la mise en réseau des conteneurs, les configurations, l’équilibrage de charge, la découverte des services, la haute disponibilité, la configuration des hôtes Docker, et bien plus encore.</span><span class="sxs-lookup"><span data-stu-id="8d945-149">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="8d945-150">Un orchestrateur gère l’exécution, la distribution, la mise à l’échelle et la réparation des charges de travail dans une collection de nœuds.</span><span class="sxs-lookup"><span data-stu-id="8d945-150">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="8d945-151">En règle générale, les produits Orchestrator sont les mêmes produits qui fournissent l’infrastructure de cluster, comme Mesosphere DC/OS, Kubernetes, Docker Swarm et Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="8d945-151">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
