<span data-ttu-id="2fcc5-103">Une page Web pouvant être affichée n’importe où dans le monde, les opérations qui analysent et mettent en forme les valeurs de date et d’heure ne doivent pas s’appuyer sur un format par défaut (généralement le format de la culture locale du serveur Web) lors de l’interaction avec l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="2fcc5-103">Because a Web page can be displayed anywhere in the world, operations that parse and format date and time values should not rely on a default format (which most often is the format of the Web server's local culture) when interacting with the user.</span></span> <span data-ttu-id="2fcc5-104">Au lieu de cela, les formulaires Web qui gèrent les chaînes de date et d’heure saisies par l’utilisateur doivent analyser les chaînes en utilisant la culture préférée de l’utilisateur.</span><span class="sxs-lookup"><span data-stu-id="2fcc5-104">Instead, Web forms that handle date and time strings input by the user should parse the strings using the user's preferred culture.</span></span> <span data-ttu-id="2fcc5-105">De même, les données de date et d’heure doivent être affichées à l’utilisateur dans un format conforme à sa culture.</span><span class="sxs-lookup"><span data-stu-id="2fcc5-105">Similarly, date and time data should be displayed to the user in a format that conforms to the user's culture.</span></span> <span data-ttu-id="2fcc5-106">Cette rubrique montre comment procéder.</span><span class="sxs-lookup"><span data-stu-id="2fcc5-106">This topic shows how to do this.</span></span>
