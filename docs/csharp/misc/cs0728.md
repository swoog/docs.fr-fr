 <span data-ttu-id="3082a-105">Il existe plusieurs scénarios où les blocs `using` ou `lock` provoquent une fuite temporaire de ressources.</span><span class="sxs-lookup"><span data-stu-id="3082a-105">There are several scenarios where `using` or `lock` blocks will result in a temporary leak of resources.</span></span> <span data-ttu-id="3082a-106">En voici un exemple :</span><span class="sxs-lookup"><span data-stu-id="3082a-106">Here is one example:</span></span>  
  
 `thisType f = null;`  
  
 `using (f)`  
  
 `{`  
  
 `f = new thisType();`  
  
 `...`  
  
 `}`  
  
 <span data-ttu-id="3082a-107">Dans ce cas, la valeur d’origine (null) de la variable `thisType` est supprimée à la fin de l’exécution du bloc `using` , mais l’objet `thisType` créé dans le bloc ne l’est pas, même s’il fait finalement l’objet d’une récupération de mémoire.</span><span class="sxs-lookup"><span data-stu-id="3082a-107">In this case, the original value, such as null, of the variable `thisType` will be disposed of when the `using` block finishes executing, but the `thisType` object created inside the block will not be, although it will eventually get garbage collected.</span></span>

## <a name="example"></a><span data-ttu-id="3082a-110">Exemple</span><span class="sxs-lookup"><span data-stu-id="3082a-110">Example</span></span>  
 <span data-ttu-id="3082a-111">Le code suivant génère l’erreur CS0728.</span><span class="sxs-lookup"><span data-stu-id="3082a-111">The following code will generate warning CS0728.</span></span>  
  
```csharp 
// CS0728.cs  
  
using System;  
public class ValidBase : IDisposable  
{  
    public void Dispose() {  }  
}  
  
public class Logger  
{  
    public static void dummy()  
    {  
        ValidBase vb = null;  
        using (vb)   
        {  
            vb = null;  // CS0728  
        }  
        vb = null;  
    }  
    public static void Main() { }  
}  
```
