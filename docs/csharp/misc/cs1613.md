# <a name="compiler-error-cs1613"></a><span data-ttu-id="6a314-102">Erreur du compilateur CS1613</span><span class="sxs-lookup"><span data-stu-id="6a314-102">Compiler Error CS1613</span></span>
<span data-ttu-id="6a314-103">La classe wrapper de coclasse managée 'class' pour l’interface 'interface' est introuvable (vous manque-t-il une référence d’assembly ?)</span><span class="sxs-lookup"><span data-stu-id="6a314-103">The managed coclass wrapper class 'class' for interface 'interface' cannot be found (are you missing an assembly reference?)</span></span>  
  
 <span data-ttu-id="6a314-104">L’utilisateur a tenté d’instancier un objet COM à partir d’une interface.</span><span class="sxs-lookup"><span data-stu-id="6a314-104">An attempt was made to instantiate a COM object from an interface.</span></span> <span data-ttu-id="6a314-105">L’interface présente les attributs **ComImport** et `CoClass` , mais le compilateur ne trouve pas le type donné par l’attribut `CoClass` .</span><span class="sxs-lookup"><span data-stu-id="6a314-105">The interface has the **ComImport** and `CoClass` attributes, but the compiler cannot find the type given for the `CoClass` attribute.</span></span>  
  
 <span data-ttu-id="6a314-106">Pour résoudre cette erreur, essayez l’une des opérations suivantes :</span><span class="sxs-lookup"><span data-stu-id="6a314-106">To resolve this error, you can try one of the following:</span></span>  
  
-   <span data-ttu-id="6a314-107">Ajoutez une référence à l’assembly qui comprend la coclasse (le plus souvent, l’interface et la coclasse doivent être dans le même assembly).</span><span class="sxs-lookup"><span data-stu-id="6a314-107">Add a reference to the assembly that has the coclass (most of the time the interface and coclass should be in the same assembly).</span></span> <span data-ttu-id="6a314-108">Pour plus d’informations, consultez [/reference](../../csharp/language-reference/compiler-options/reference-compiler-option.md) ou [Ajouter une référence, boîte de dialogue](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) .</span><span class="sxs-lookup"><span data-stu-id="6a314-108">See [/reference](../../csharp/language-reference/compiler-options/reference-compiler-option.md) or [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) for information.</span></span>  
  
-   <span data-ttu-id="6a314-109">Corrigez l’attribut `CoClass` dans l’interface.</span><span class="sxs-lookup"><span data-stu-id="6a314-109">Fix the `CoClass` attribute on the interface.</span></span>  
  
 <span data-ttu-id="6a314-110">L’exemple suivant montre une utilisation correcte de **CoClassAttribute**:</span><span class="sxs-lookup"><span data-stu-id="6a314-110">The following sample demonstrates correct usage of **CoClassAttribute**:</span></span>  
  
```csharp  
// CS1613.cs  
using System;  
using System.Runtime.InteropServices;  
  
[Guid("1FFD7840-E82D-4268-875C-80A160C23296")]  
[ComImport()]  
[CoClass(typeof(A))]  
public interface IA{}  
public class A : IA {}  
  
public class AA  
{  
   public static void Main()  
   {  
      IA i;  
      i = new IA(); // This is equivalent to new A().  
                    // because of the CoClass attribute on IA  
   }  
}  
```
