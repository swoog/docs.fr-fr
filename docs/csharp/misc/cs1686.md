# <a name="compiler-error-cs1686"></a><span data-ttu-id="34c62-102">Erreur du compilateur CS1686</span><span class="sxs-lookup"><span data-stu-id="34c62-102">Compiler Error CS1686</span></span>
<span data-ttu-id="34c62-103">L’adresse de la variable locale 'variable' ou de ses membres ne peut pas être prise et utilisée dans une méthode anonyme ou une expression lambda</span><span class="sxs-lookup"><span data-stu-id="34c62-103">Local 'variable' or its members cannot have their address taken and be used inside an anonymous method or lambda expression</span></span>  
  
 <span data-ttu-id="34c62-104">Cette erreur est générée lorsque vous utilisez une variable, que vous tentez de prendre son adresse et que l’une de ces actions s’effectue à l’intérieur d’une méthode anonyme.</span><span class="sxs-lookup"><span data-stu-id="34c62-104">This error is generated when you use a variable, and attempt to take its address, and one of these actions is done inside an anonymous method.</span></span>  
  
## <a name="example"></a><span data-ttu-id="34c62-105">Exemple</span><span class="sxs-lookup"><span data-stu-id="34c62-105">Example</span></span>  
 <span data-ttu-id="34c62-106">L’exemple suivant génère l’erreur CS1686.</span><span class="sxs-lookup"><span data-stu-id="34c62-106">The following sample generates CS1686.</span></span>  
  
```csharp  
// CS1686.cs  
// compile with: /unsafe /target:library  
class MyClass  
{  
   public unsafe delegate int * MyDelegate();  
  
   public unsafe int * Test()  
   {  
      int j = 0;  
      MyDelegate d = delegate { return &j; };   // CS1686  
      return &j;   // OK  
   }  
}  
```
